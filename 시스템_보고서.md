## 시스템 개요

### 목적
이 시스템은 **텍스트를 입력하면 AI 아바타가 말하는 영상을 자동으로 생성**하는 웹 애플리케이션입니다. 사용자는 간단한 웹 인터페이스를 통해 텍스트를 입력하고, 시스템이 음성 합성(TTS)과 비디오 생성 기술을 결합하여 최종 영상을 만들어냅니다.

### 핵심 기능
- **텍스트 → 음성 변환**: 입력된 텍스트를 자연스러운 한국어 음성으로 변환
- **음성 → 영상 생성**: 생성된 음성에 맞춰 AI 아바타의 입 모양을 동기화하여 영상 생성
- **영상 편집**: 생성된 영상의 크롭, 해상도 조정, 투명 배경 처리 등 후처리 기능
- **다양한 출력 형식**: 일반 MP4 영상 또는 투명 배경(알파 채널) 포함 영상 생성

### 사용자 대상
- 마케팅 콘텐츠 제작자
- 교육 콘텐츠 제작자
- 금융 서비스 안내 영상 제작자
- 기타 비디오 콘텐츠 제작이 필요한 사용자

### 기술적 차별성 및 기대효과

#### 기술적 차별성
- **End-to-End 파이프라인 구현**: 텍스트 입력부터 영상 출력까지 통합 처리
- **고품질 립싱크 기술**: 음성에 맞춰 자연스러운 입 모양 동기화
- **다양한 출력 형식 지원**: 일반 영상 및 투명 배경 영상 생성
- **실시간 후처리 기능**: 크롭, 해상도 조정 등 즉시 편집 가능
- **프리셋 기반 워크플로우**: 자주 사용하는 설정 저장 및 재사용

#### 기대효과
- **콘텐츠 제작 효율성 향상**: 전문 영상 편집 지식 없이도 고품질 영상 제작 가능
- **일관된 품질 보장**: AI 기반 자동 생성으로 품질 편차 최소화
- **비용 절감**: 전문 제작 인력 및 장비 없이도 영상 콘텐츠 제작 가능
- **빠른 제작 속도**: 텍스트 입력 후 수분 내 영상 생성 완료
- **확장 가능성**: 다양한 채널(웹/모바일/키오스크)에 적용 가능한 기반 확보

```
               │                               │

                               │ 생성된 파일 반환
                               ▼

                    │  - 생성된 영상 저장     │
                    │  - 오디오 파일 저장     │
                    └──────────────────────┘
```

### 아키텍처 특징

1. **분리형 아키텍처 (Microservices)**
   - Web Demo, TTS 서버, 비디오 생성 서버가 각각 독립적으로 운영
   - 각 서버는 특정 기능에만 집중하여 유지보수성 향상
   - 서버 간 HTTP API로 통신하여 확장성 확보

2. **비동기 처리**
   - FastAPI의 비동기 기능을 활용하여 여러 요청 동시 처리 가능
   - 비디오 생성은 시간이 오래 걸리므로 스트리밍 방식으로 파일 수신

3. **파일 관리**
   - 임시 파일은 `temp/` 디렉토리에 저장 후 처리 완료 시 삭제
   - 최종 결과물은 `outputs/` 디렉토리에 영구 저장
   - UUID를 사용하여 파일명 충돌 방지

## 모듈별 상세 설명

### 1. Web Demo 서버 (app.py)

#### 핵심 메시지
사용자 요청을 받아 TTS와 비디오 생성 서버를 조율하고, 생성된 영상의 후처리까지 담당하는 통합 웹 애플리케이션 서버를 구축하였다.

#### 역할
- 사용자 인터페이스 제공 및 요청 처리
- TTS 서버와 비디오 생성 서버 간의 조율
- 생성된 파일의 후처리 및 관리

#### 개발 목표
- 사용자가 텍스트만 입력하면 자동으로 영상이 생성되는 원클릭 워크플로우 제공
- 여러 서버를 조율하여 안정적이고 빠른 응답 제공
- 생성된 영상에 대한 즉시 편집 기능 제공

#### 주요 모듈

##### 1.1 웹 인터페이스 제공
```python
@app.get("/", response_class=HTMLResponse)
async def index(request: Request):
    """메인 페이지 렌더링"""
    return templates.TemplateResponse("index.html", {"request": request})
```
- 사용자에게 웹 인터페이스 제공
- HTML 템플릿을 렌더링하여 반환

##### 1.2 음성 생성 API 호출
```python
@app.post("/api/tts")
async def generate_tts(...):
    """TTS 서버에 음성 생성 요청"""
```
- 텍스트와 음성 설정을 TTS 서버로 전달
- 생성된 오디오 파일을 임시 저장

**처리 흐름:**
1. 사용자 입력 (텍스트, 음성 모델, 속도 등) 수신
2. TTS 서버에 HTTP POST 요청 전송
3. 오디오 파일 수신 및 임시 저장
4. 파일 경로 반환

##### 1.3 비디오 생성 API 호출
```python
@app.post("/api/generate_video")
async def generate_video_api(...):
    """TTS + 비디오 생성 통합 처리"""
```
- 음성 생성과 비디오 생성을 순차적으로 처리
- 오디오만 생성하거나 영상까지 생성하는 옵션 제공

**처리 흐름:**
1. **음성 생성 단계**
   - 텍스트를 TTS 서버로 전송
   - 생성된 오디오 파일을 `temp/` 디렉토리에 저장

2. **비디오 생성 단계** (오디오만 생성 시 생략)
   - 오디오 파일을 비디오 생성 서버로 전송
   - 아바타 ID, 립싱크 설정 등 파라미터 전달
   - 생성된 비디오를 스트리밍 방식으로 수신
   - 브라우저 호환 포맷(H.264)으로 재인코딩

3. **후처리 단계**
   - 투명 배경 영상 생성 (선택 시)
   - 파일을 `outputs/` 디렉토리로 이동
   - 임시 파일 정리

##### 1.4 영상 크롭 및 편집
```python
@app.post("/api/crop_video")
async def crop_video(req: CropRequest):
    """영상 크롭 및 해상도 조정"""
```
- 생성된 영상의 특정 영역을 잘라내기
- 출력 해상도 조정
- FFmpeg를 사용하여 비디오 처리

**기능:**
- 크롭 영역 지정 (X, Y, 너비, 높이)
- 출력 해상도 설정
- 화면 비율 고정 (1:1, 16:9, 9:16 등)
- 프리셋 저장 및 불러오기

##### 1.5 프리셋 관리
```python
@app.get("/api/crop_presets")
@app.post("/api/crop_presets")
```
- 크롭 설정을 프리셋으로 저장
- 저장된 프리셋을 불러와 재사용
- 프리셋 내보내기/가져오기 기능

### 3. TTS 서버 (외부 서버)

#### 핵심 메시지
실서비스 적용 수준의 자연스러운 한국어 음성 합성 기능을 제공하여, 금융 서비스에 적합한 신뢰감 있는 음성을 생성한다.

#### 역할
- 텍스트를 음성으로 변환
- 한국어 음성 합성 모델 실행

#### 개발 목표
- 금융 서비스에 적합한 전문적이고 신뢰감 있는 음성 제공
- 다양한 상황(대화/발표)에 맞는 음성 톤 선택 가능
- 실시간 처리에 적합한 빠른 음성 생성 속도 확보

#### 주요 기능
- **음성 모델**: 
  - `pds_natural`: 자연스러운 대화 톤
  - `pds_announcer`: 방송/뉴스 톤
- **설정 옵션**:
  - 발화 속도 (0.5x ~ 1.2x)
  - 시작/종료 여백
  - 문장 사이 여백

#### API 인터페이스
```
POST http://localhost:7009/tts
Content-Type: application/json

{
    "text": "안녕하세요",
    "model": "pds_natural",
    "sr": 16000,
    "speed": 1.0,
    "pre_post_silence_sec": 3.0,
    "intermittent_silence_sec": 0.6
}

Response: WAV 오디오 파일 (바이너리)
```

### 4. 비디오 생성 서버 (외부 서버)

#### 핵심 메시지
음성에 맞춰 자연스럽게 입 모양이 움직이는 AI 아바타 영상을 생성하는 핵심 기술을 제공한다.

#### 역할
- 오디오 파일을 받아 AI 아바타 영상 생성
- 립싱크(Lip Sync) 처리

#### 개발 목표
- 음성과 입 모양이 정확하게 동기화된 자연스러운 영상 생성
- 발화 시작/종료 구간의 부드러운 전환 제공
- 다양한 아바타 지원으로 활용 범위 확대

#### 주요 기능
- **립싱크**: 오디오에 맞춰 입 모양 동기화
- **아바타 선택**: 다양한 AI 아바타 지원
- **부드러운 전환**: 프레임 간 자연스러운 전환

#### API 인터페이스
```
POST http://localhost:8001/synthesize
Content-Type: multipart/form-data

Files:
  - audio_file: WAV 파일

Data:
  - avatar_id: "PDS_blue_smooth5_margin10_bboxshift10_v2"
  - audio_transition_frames: 4
  - frame_transition_frames: 3
  - interpolation_method: "cosine"
  - pad_start_frames: 10
  - pad_end_frames: 10
  - fps: 25
  - batch_size: 8

Response: MP4 비디오 파일 (스트리밍)
```

### 사용자 흐름

#### 1. 초기 화면 (대기 상태)
- 좌측: 입력 폼 표시
- 중앙: 안내 메시지 및 시스템 소개
- 우측: 크롭 패널 비활성화

#### 2. 입력 및 설정
1. **말투 선택**: 자연 톤 또는 방송 톤 선택
2. **생성 방식 선택**:
   - 영상 생성: 아바타가 말하는 영상 생성
   - 음성만 생성: 오디오 파일만 생성
3. **파일 형식 선택** (영상 생성 시):
   - MP4: 일반 영상
   - 투명 배경 포함: 알파 채널이 있는 영상
4. **아바타 선택** (영상 생성 시)
5. **텍스트 입력**: 발화할 내용 입력 (예시 버튼 제공)
6. **고급 설정** (선택):
   - 음성 속도 조정
   - 입모양 자연스러움 설정
   - 타이밍 조정

#### 3. 생성 요청
- "영상 생성하기" 또는 "음성 생성하기" 버튼 클릭
- 버튼 비활성화 (중복 요청 방지)

#### 4. 처리 중 (로딩 상태)
- 진행률 바 표시
- 단계별 상태 메시지:
  - "음성 생성 중..."
  - "영상 생성 중..." (영상 생성 시)
  - "마무리 중..."
- 예상 소요 시간 표시

#### 5. 결과 표시
**음성만 생성 시:**
- 오디오 플레이어 표시
- 다운로드 버튼 제공
- 파일 크기 정보 표시

**영상 생성 시:**
- 비디오 플레이어 표시
- 크롭 패널 활성화
- 다운로드 버튼 제공
- 파일 정보 표시

#### 6. 영상 편집 (선택)
1. **크롭 영역 지정**:
   - 비디오 위에서 드래그하여 영역 선택
   - 또는 수치로 직접 입력
2. **화면 비율 선택**: 자유 / 1:1 / 16:9 / 9:16
3. **출력 해상도 설정**: 프리셋 또는 커스텀
4. **미리보기 확인**: 우측 패널에서 실시간 미리보기
5. **적용**: 크롭된 영상 생성
6. **프리셋 저장** (선택): 설정을 저장하여 재사용

#### 7. 다운로드
- 생성된 파일 다운로드
- "다시 생성" 버튼으로 새 요청 가능

### 테마 전환
- 우측 상단 버튼으로 다크 모드 / 라이트 모드 전환
- 설정은 브라우저 로컬 스토리지에 저장

## 기술 스택

### 백엔드
- **FastAPI** (0.109.0): 고성능 비동기 웹 프레임워크
- **Uvicorn** (0.27.0): ASGI 서버
- **Python 3.10**: 프로그래밍 언어
- **Requests**: HTTP 클라이언트 (외부 API 호출)
- **Jinja2**: 템플릿 엔진 (HTML 렌더링)
- **aiofiles**: 비동기 파일 I/O

### 프론트엔드
- **HTML5**: 마크업
- **CSS3**: 스타일링 (커스텀 다크/라이트 테마)
- **JavaScript (Vanilla)**: 클라이언트 사이드 로직
- **Tailwind CSS**: 유틸리티 CSS 프레임워크 (오프라인 버전)

### 비디오 처리
- **FFmpeg**: 비디오 인코딩/디코딩, 크롭, 포맷 변환
- **FFprobe**: 비디오 메타데이터 추출

### 파일 형식
- **입력**: 텍스트 (UTF-8)
- **중간 파일**: WAV 오디오 (16kHz, 모노)
- **출력**: 
  - MP4 (H.264 비디오, AAC 오디오)
  - MOV/WEBM (투명 배경, 알파 채널)

### 개발 도구

- **Docker Compose**: 다중 컨테이너 관리

```
   ↓
2. Web Demo 서버 (app.py)
   ↓

   ↓

   POST http://localhost:7009/tts
   {
     "text": "...",
     "model": "pds_natural",

     ...
   }
   ↓
   [WAV 오디오 파일 수신]
   ↓

   temp/audio_{uuid}.wav
   ↓

   POST http://localhost:8001/synthesize

   ↓
   [MP4 비디오 파일 스트리밍 수신]
   ↓
6. 비디오 후처리
   - 브라우저 호환 포맷 변환 (H.264)
   - 투명 배경 처리 (선택 시)
   ↓

   outputs/video_{uuid}.mp4
   outputs/audio_{uuid}.wav (음성만 생성 시)
   ↓
8. 응답 반환
   {
     "success": true,
     "video_url": "/outputs/video_xxx.mp4",
     "video_id": "...",
     ...
   }
   ↓
9. 프론트엔드 처리
   - 비디오 플레이어에 URL 설정
   - 다운로드 링크 생성
   - 크롭 패널 활성화

```

```
   ↓
   [X, Y, 너비, 높이, 출력 해상도]
   ↓
2. 프론트엔드에서 미리보기 표시
   - Canvas에 크롭된 영역 그리기
   - 실시간 업데이트
   ↓

   ↓

   {
     "source_url": "/outputs/video_xxx.mp4",
     "x": 100,
     "y": 50,
     "width": 800,
     "height": 600,
     "output_width": 1920,
     "output_height": 1080
   }
   ↓
5. FFmpeg로 비디오 처리
   ffmpeg -i source.mp4 -vf "crop=800:600:100:50,scale=1920:1080" output.mp4
   ↓
6. 크롭된 영상 저장
   outputs/crop_{uuid}.mp4
   ↓
7. 다운로드 링크 업데이트
```

## 주요 기능 상세

### 1. 음성 생성 기능

#### 지원 음성 모델
- **자연 톤 (pds_natural)**: 일상 대화에 적합한 자연스러운 목소리
- **방송 톤 (pds_announcer)**: 뉴스나 발표에 적합한 공식적인 목소리

#### 음성 설정
- **발화 속도**: 0.5x (느림) ~ 1.2x (빠름), 기본값 1.0
- **시작/종료 여백**: 문장 시작 전과 끝난 후의 무음 구간 (기본 3.0초)
- **문장 사이 여백**: 문장 사이의 짧은 쉼 (기본 0.6초)

### 2. 영상 생성 기능

#### 립싱크 설정
- **입 움직임 부드러움 (audio_transition_frames)**: 
  - 입 열림/닫힘 전환 시 부드러움 정도
  - 권장값: 4~5 프레임
- **영상 블렌딩 부드러움 (frame_transition_frames)**:
  - 화면 전환의 매끄러움
  - 권장값: 3 프레임
- **부드러움 방식 (interpolation_method)**:
  - `cosine`: 자연스러운 곡선 (추천)
  - `linear`: 일정한 속도
  - `sigmoid`: 부드러운 S자 곡선
  - `ease-in-out`: 천천히 시작/종료

#### 타이밍 조정
- **말하기 시작 전 대기 (pad_start_frames)**: 
  - 발화 직전 준비 동작
  - 권장값: 8~10 프레임
- **말하기 종료 후 대기 (pad_end_frames)**:
  - 발화 후 마무리 동작
  - 권장값: 4~10 프레임

### 3. 출력 형식

#### 일반 MP4
- **코덱**: H.264 (비디오), AAC (오디오)
- **용도**: 일반적인 영상 재생
- **호환성**: 모든 브라우저 및 플레이어 지원

#### 투명 배경 포함
- **MOV (알파)**:
  - 코덱: qtrle (기본), prores_ks 4444, png
  - 용도: 전문 편집 소프트웨어에서 사용
- **WEBM (알파)**:
  - 코덱: VP9 (비디오), Opus (오디오)
  - 용도: 웹에서 투명 배경 영상 재생
- **미리보기**: MOV 형식의 경우 브라우저 호환을 위해 WEBM 미리보기 자동 생성

### 4. 크롭 및 편집 기능

#### 크롭 방법
1. **드래그 앤 드롭**: 비디오 위에서 마우스로 영역 선택
2. **수치 입력**: X, Y, 너비, 높이를 직접 입력
3. **핸들 조정**: 크롭 박스의 모서리 핸들을 드래그하여 크기 조정

#### 화면 비율
- **자유**: 원하는 비율로 자유롭게 크롭
- **1:1**: 정사각형 (SNS용)
- **16:9**: 와이드스크린 (일반 영상)
- **9:16**: 세로형 (모바일/쇼츠)

#### 출력 해상도
- **현재 프레임 유지**: 크롭된 영역의 원본 해상도 유지
- **프리셋 해상도**: 
  - 1:1: 1024×1024, 2048×2048
  - 16:9: 1280×720, 1920×1080
  - 9:16: 720×1280, 1080×1920
- **커스텀**: 원하는 해상도 직접 입력
  - 가로 픽셀 또는 스케일 비율로 설정
  - 세로는 자동 계산

### 5. 프리셋 관리

#### 프리셋 저장
- 크롭 설정을 이름과 함께 저장
- 이름 미입력 시 자동으로 타임스탬프 기반 이름 생성
- 최대 20개까지 저장 (초과 시 오래된 것부터 삭제)

#### 프리셋 불러오기
- 저장된 프리셋 목록에서 선택하여 불러오기
- "최근 설정 불러오기": 마지막으로 적용한 설정 복원

#### 프리셋 공유
- **내보내기**: 모든 프리셋을 JSON 파일로 다운로드
- **가져오기**: JSON 파일을 업로드하여 프리셋 추가

### 6. 에러 처리

#### 연결 오류
- TTS 서버 연결 실패: "음성 생성 서버에 연결할 수 없습니다"
- 비디오 생성 서버 연결 실패: "비디오 생성 서버에 연결할 수 없습니다"
- 해결 방법 안내 메시지 표시

#### 타임아웃
- 비디오 생성 시간 초과: "요청 시간 초과" 메시지
- 사용자에게 재시도 안내

#### 파일 오류
- 비디오 파일이 생성되지 않음
- 비디오 파일이 비어있음
- FFmpeg 처리 실패
